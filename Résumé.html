Les TICs regroupent l'ensemble des outils mis en place pour manipuler, traiter (matériels et logiciels) , produire, modifier et échanger(réseaux et moyens de transmission), permettre une meilleure diffusion de l'information, de conserver(les supports de stockage) et présenter sous forme vocale, texte, image ou des données contenues dans des signaux en utilisant les techniques de l'information et internet, l'audiovisuel, Multimédia et des télécommunications.

    • CHAPITRE 03

    Le Modèle OSI

    1/ Les composants matériel d'un Réseau informatique:

    • Les cartes réseau :

    appelées cartes d'interface réseau (NIC), sont installées à l'intérieur de l'ordinateur. Ils permettent à l'ordinateur de se connecter à un réseau et aussi de transmettre et recevoir des données via des protocoles de communication spécifiques.

    • SWITCH :

    Un switch (ou commutateur réseau) est utilisé pour connecter des appareils sur un réseau local (LAN)et leurs permettre de communiquer en utilisant des adresses MAC (Media Access Control) avec toute sécurité. Les switches sont disponibles en différentes tailles commençant des petits switches pour les réseaux domestiques, aux switches de niveau entreprise pour les réseaux plus importants.

    • HUB :

    ou concentrateur en français, est utilisé pour connecter plusieurs appareils sur un réseau local (LAN). Ce composant matériel reçoit les données envoyées par un appareil et les transmet à tous les autres appareils connectés. La différence entre le HUBS et le SWITCH est que le SWITCH utilise une table de correspondance pour envoyer les paquets uniquement aux nœuds destinataires, alors que le HUB envoie les paquets a tous les nœuds connectées, c'est pour cela ils ont été remplacés par les switches.


    2/ Le Modèle OSI:

    est un modèle de référence pour les communications de réseau qui divise les communications en sept couches distinctes. Chaque couche est responsable de fonctions spécifiques liées à la communication de données entre les appareils de réseau. Les sept couches sont la couche physique, la couche liaison de données, la couche réseau, la couche transport, la couche session, la couche présentation et la couche application. Bien que le modèle OSI ne soit pas utilisé dans les protocoles de communication de réseau modernes, il reste utile pour comprendre les principes fondamentaux de la communication de réseau.


    3/ Le Modèle TCP/IP:

    Le modèle TCP/IP (Transmission Control Protocol/Internet Protocol) est un modèle de référence pour les communications de réseau utilisé pour la conception et la mise en œuvre des réseaux informatiques modernes. Ce modèle divise les communications en quatre couches principales : la couche liaison de données, la couche réseau, la couche transport et la couche application. Le modèle TCP/IP est utilisé pour la conception et la mise en œuvre des réseaux d'entreprise, des réseaux Internet et des réseaux sans fil.

    4/ L'adressage:

    Les communications dans un réseau suivent 3 ces règles : l'adressage, l'architecture et le protocole.

    L'adressage IP: (Internet Protocol) c'est le numéro qui identifie chaque ordinateur dans les réseaux.

    1/ ADRESSE IPV4 :

    Qui Utilise une représentation binaire compose de 32 bits exprimés en décimale et séparée par des points(.).

    2/ ADRESSE IPV6 :

    Qui utilise des adresses de 128bits représentée en hexadécimale séparée par deux points[:].

    6\ Créer un dépôt Git à partir d'un répertoire existant:

    1\ On démarre avec git ( les projet en cours sont déjà stoker dans la machine).

    2\ Créer un répertoire contient des fichier de text vide.

    3\ Initialiser le dépôt git en utilisant la commande 'git unit'(le dépôt est vide) et utiliser la commande 'git status' pour déterminer l'état des fichiers de répertoire.

    4\ Indexer les fichiers par utilisation de la commande 'git add'. (utiliser le nom de fichier pour indexer le fichier et le nom de répertoire pour indexer tous)

    5\ Utiliser la commande 'git commit' pour valider les fichiers et les ajouter en base .

    6\ Effectuer le commit en ajoutant le message .

    7\ Après , on utilise à nouveau 'git status' , on constate que tous les fichier sont suivis et enregistrés dans le dépôt et aucune modification n'a été apportée.

    On va devoir utiliser les commandes 'git add' et 'git commit'a chaque fois qu'on veux enregistrer une modification ou ajouter un fichiers.

    Effectuer un 'git add' juste avant un 'git commit' pour enregistrer toujours la dernière version de fichier.
